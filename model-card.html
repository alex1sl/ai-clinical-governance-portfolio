Model Card: HEDIS Asthma Logic Validator

1. Model Details

  • Governance Lead: Alexis Smith
  • Base Model: GPT-4.5
  • Version: 1.0 (NIST RMF 1.0 Aligned)
  • Audit Focus: HEDIS MY 2026 Compliance

2. Intended Use & Compliance Measures

This model is an Audit Support Tool used to verify if patients diagnosed with Asthma in Outpatient, Urgent Care, or Hospital settings received a 30-day follow-up encounter.

HEDIS Success Criteria:
  • MET: Follow-up scheduled/seen within ≤ 30 days of discharge or initial visit.
  • NOT MET: Encounter date > 30 days or no follow-up identified.

3. Date Calculation Logic (Evaluation Data)

Red-teaming was performed to test the model's ability to handle complex calendar logic without "hallucinating" dates.

  • Test Case 1: Leap Year transitions (Feb 29).
  • Test Case 2: Non-business days/Holiday window expirations.
  • Test Case 3: Cross-setting date extraction (Urgent Care vs. Hospital).

4. Key Metrics & Risk Mitigation

Risk Identified Safety Guard (NIST Control)
Chronological Drift Threshold Logic: Automated flag for any date > 28 days for mandatory human review.
Logic Hallucination Source Citation: Model is required to cite the specific data point used for calculation.
Reporting Error SQL Cross-Validation: Queries audit AI output against raw dataset timestamps.

5. Governance Conclusion

This model is strictly for administrative governance to ensure the highest quality measures are met. It does not provide clinical diagnosis or autonomous decision-making.
