Deep Dive: HEDIS Asthma 30-Day Follow-Up

Data Integrity & Chronological Logic

The core evaluation focused on testing the AI's ability to accurately calculate the 30-day follow-up window mandated by HEDIS MY 2026. Ensuring chronological accuracy is vital for meeting quality benchmarks.

  • The Challenge: The model failed to maintain accurate date logic on edge cases, such as Leap Years, resulting in dates outside the compliance window.
  • The SQL Control: I utilized SQL to audit the AI-generated the follow up date against the dischage date to identify any chronological discrepancies.

Governance Protocol (HITL)

Using the NIST RMF 1.0 framework, I established safety buffers to mitigate these logic risks:

28-Day Flag

Any AI-generated date exceeding 28 days triggers a Human-in-the-Loop (HITL)

Red-Teaming Strategy

Tested specifically for Leap Year sequences to expose and document model logic failures in time-sensitive workflows.
